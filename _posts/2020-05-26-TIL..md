### 📚 웹 사이트와 웹 어플리케이션의 차이점

|         웹사이트          |                                              웹 어플리케이션                                               |
| :-----------------------: | :--------------------------------------------------------------------------------------------------------: |
| 정보 제공만 가능 (쌍방향) | 정보 제공을 포함하여 방문자도 무언가를 해야할 때 (양방향) ex) <br>사용자 계정, 반응, 사용자 지정, CRM 통합 |
|    html,css,js로 구성     |                 ASP,JSP,PHP 같은 서버 사이드 스크립트 언어가 필요하다. angular,mern로 구현                 |
|                           |                                                                                                            |

- 웹 앱은 유저이탈을 막을 수 있다.  
  **(why)**
  1. 웹 앱을 통해 유저에게 푸시 알람 요청을 할 수 있음 👉서비스 업데이트에 유저를 부를 수 있다
  2. 모바일 앱 개발하지 않아도 모바일 앱의 기능을 웹 브라우저상에서도 거의 동일하게 실현이 가능하다. (예산절감)
  3. 모바일 앱처럼 직접 기기에 접근 허용등을 요청하지 않기 때문에 보안 면에서 모바일 앱보다 좋다.
  4. 개인 맞춤형 서비스 제공이 가능하다 ex) 아마존, 스포티파이에서 사용자 화면을 권장사항으로 확산

#### 📖PWA ( Progressive Web Apps)

웹 브라우저에서 실행되는 웹 응용 프로그램. 구글에서 밀어주고 있다. seo 친화적, 반응형 디자인 적용되어 있음

1. 오프라인 작업 가능해 데이터 사용량이 낮다.
2. 푸시 알림과 디바이스의 화면에 아이콘이 있어 유저들의 접근을 높여준다.
3. 보안이 좋다 : https로 연결이 되고, 웹 url로만 접근하면 된다. (기존의 네이티브 앱의 경우 설치 시간이 걸려 유저들이 이탈하게 됨)
4. 로딩시간이 빠르다.
