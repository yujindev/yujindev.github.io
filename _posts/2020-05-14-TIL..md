## 알게 된 것 📝

#### 📚async vs defer

1.  **head에 넣는다**  
    → 페이지 로딩 속도 느려짐
2.  **body끝에 넣는다**  
    → 사용자가 이미 콘텐츠를 보고 있지만, js에 의존적인 페이지라면 의도하는 바를 실현하기가 어렵다. 서버에서 js를 받고 실행하는 시간이 필요함
3.  **head + async**  
    script 값에 async를 선언함  
    async는 boolean이어서 선언하는 것만으로 true로 설정 됨  
    브라우저가 html을 다운받으면서 병렬로 다운받게 된다.  
    다운로드 시간을 절약할 수 있다.  
    js로 조작하려는 요소가 정의되지 않은 상태에서 실현되는 가능성, js를 실현하고 나머지 html를 다운로드하기때문에 여전히 사용자에게는 느리다.  
    ✨여러 스크립트를 다운받게 될 경우, 정의 된 스크립트 순서와는 관계없이 다운로드순으로 실행이 된다.

4.  **_head+defer_**  
    html과 js를 병렬로 동시에 다운받고, html이 다 다운받은 후에 js가 실행된다.
    html을 parsing하는 동안 스크립트들을 병렬적으로 다운로드 한다.

#### 📚'use strict';

바닐라 js로 개발할 때는 js파일 **_맨 위에 'use strict'을 기재할 것_**  
js는 굉장히 유연하다. → 때로는 위험하다. → 실수 확률이 높다. &nbsp;
선언되지 않은 변수에 값을 할당한다던가 기존의 프로토타입을 변경등의 비상식적인 것들을 쓰지 않도록 해준다. → 에러를 확인이 가능하다.  
js엔진이 효율적으로 js를 확인할 수 있어 성능 발전도 할 수 있다.

## 회고 🌿

지금까지 스크립트를 걸때는 바디 끝에만 걸었었는데 async와 defer라는 새로운 방법을 알게 되었다. 기존에 배포했었던 페이지가 너무 무거워서 js 걸었던 것들을 다 지웠었던 것을 생각하면 아깝기도 하고, 그 때 오늘 안 것을 알았다면 어느정도 속도가 높아졌을까 라는 생각이 든다. 웹 페이지는 로딩싸움이기 때문에 더 빨리 더 가볍게! 라는 것에 집중하고 있다. 오늘의 배움을 발판으로 삼아 현명하게 코드를 짤 수 있는 사람이 되어야겠다고 다짐한다.
